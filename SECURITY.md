# Security Policy

## Supported Versions

Only the latest version deployed to production is supported with security updates.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly. **Do not open a public GitHub issue.**

Instead, please email **security@samduke.dev** with:

- A description of the vulnerability
- Steps to reproduce the issue
- Any relevant logs or screenshots

You should receive a response within 72 hours. Once the issue is confirmed, a fix will be prioritized and released as soon as possible.

## Scope

The following are in scope:

- Authentication and session handling
- GitHub token encryption and storage
- API token validation
- Permission enforcement
- Content Security Policy bypasses
- Path traversal or injection attacks

## Out of Scope

- Vulnerabilities in third-party dependencies (please report these upstream)
- Issues requiring physical access to a user's device
- Social engineering attacks
